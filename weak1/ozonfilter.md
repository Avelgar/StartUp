# Проект Ozon Filter
## 1. Описание идеи

Расширение для браузера Chrome, предназначенное для фильтрации товаров на площадке Ozon по пользовательским правилам.

### Функциональные возможности:

- Фильтрация товаров по ключевым словам
- Исключение товаров по стоп-словам
- Интеллектуальная фильтрация изображений с помощью AI
- Пользовательские правила фильтрации
- Автоматическое скрытие нерелевантных товаров

**Основное позиционирование** — улучшение качества поиска товаров на Ozon для более эффективного шопинга.

---

## 2. Анализ рынка

### 2.1. Проблематика

- Низкое качество поиска на маркетплейсах — много нерелевантных товаров в выдаче
- Встроенные фильтры Ozon ограничены и не всегда эффективны
- Продавцы используют манипулятивные названия и теги для попадания в выдачу
- Сложность поиска конкретного товара среди тысяч похожих предложений
- Отсутствие возможности фильтрации по содержимому изображений

### 2.2. Целевые сегменты

#### Приоритетный сегмент:
**Активные покупатели маркетплейсов** (power shoppers)
- Регулярно совершают покупки на Ozon
- Тратят много времени на поиск нужных товаров
- Фрустрированы качеством поиска
- Технически подкованы для установки расширения

#### Вторичные сегменты:
1. **Барахольщики и охотники за скидками** — ищут конкретные товары по выгодным ценам
2. **Профессиональные закупщики** — покупают товары для бизнеса
3. **Селлеры** — анализируют конкурентов и рынок

### 2.3. Конкурентная среда

#### Прямые конкуренты:

| Продукт | Ценностное предложение | Характеристики | Недостатки |
|---------|------------------------|----------------|------------|
| **Встроенные фильтры Ozon** | Базовая фильтрация товаров | Бесплатно, встроено | Ограниченная функциональность, нет AI |
| **Расширения для price tracking** | Отслеживание цен | Keepa, CamelCamelCamel | Не решают проблему релевантности поиска |

**Примечание:** Специализированных AI-фильтров для маркетплейсов практически не существует — это свободная ниша.

#### Замещающие решения:

- **Ручная фильтрация** — просмотр всех товаров вручную (медленно)
- **Использование специфичных поисковых запросов** — требует опыта
- **Переход на другие маркетплейсы** — Wildberries, Яндекс.Маркет
- **Прямая покупка у продавцов** — минуя маркетплейсы

### 2.4. Драйверы развития рынка

✅ **Положительные факторы:**
1. Рост рынка e-commerce и маркетплейсов в России
2. Увеличение количества товаров на Ozon (сложнее найти нужное)
3. Развитие AI-технологий для анализа изображений (CLIP, Vision LLM)
4. Фрустрация пользователей от низкого качества поиска
5. Простота распространения браузерных расширений
6. Потенциал вирусного распространения через word-of-mouth

### 2.5. Барьеры развития рынка

❌ **Ограничения:**
1. Изменения DOM структуры сайта Ozon могут ломать расширение
2. Возможные ограничения со стороны маркетплейса (блокировка расширения)
3. Высокая нагрузка при анализе изображений (нужен backend или локальная обработка)
4. Зависимость от одной платформы (только Ozon)
5. Сложность монетизации браузерного расширения
6. Конкуренция с будущими улучшениями поиска Ozon

## 3. Предварительные выводы

### 3.1. Выбор приоритетного сегмента

**Целевой сегмент:** Активные покупатели маркетплейсов (power shoppers)

**Обоснование:**
- Высокая частота использования Ozon
- Острая боль от некачественного поиска
- Готовность устанавливать расширения для улучшения опыта
- Потенциал стать амбассадорами продукта

### 3.2. Гипотезы ценности продукта

#### Гипотеза 1: Экономия времени
**Предположение:** Пользователи готовы установить расширение, если оно экономит значительное время на поиск нужных товаров (10-15 минут на каждый поиск).

**Проверка:** A/B тестирование времени поиска, опросы пользователей

#### Гипотеза 2: AI-фильтрация изображений
**Предположение:** Возможность фильтрации товаров по содержимому изображений (например, "показать только красные кроссовки") создаёт уникальную ценность.

**Проверка:** Прототипирование, user testing

#### Гипотеза 3: Пользовательские правила
**Предположение:** Пользователи хотят создавать собственные правила фильтрации (стоп-слова, обязательные слова), которые сохраняются между сессиями.

**Проверка:** Customer development, анализ паттернов использования

#### Гипотеза 4: Вирусное распространение
**Предположение:** Решение конкретной боли приведёт к органическому вирусному распространению среди пользователей Ozon без больших маркетинговых затрат.

**Проверка:** Метрики вирального коэффициента (K-factor), NPS

---

## 4. Основные риски

⚠️ **Технические риски:**
- Изменения DOM структуры сайта Ozon могут ломать функциональность
- Высокая нагрузка при анализе изображений с помощью AI
- Сложность оптимизации производительности расширения
- Проблемы с обработкой динамического контента (infinite scroll)

⚠️ **Рыночные риски:**
- Возможные ограничения или блокировка со стороны Ozon
- Улучшение встроенного поиска Ozon может снизить ценность продукта
- Зависимость от одной платформы

⚠️ **Бизнес-риски:**
- Сложность монетизации браузерного расширения
- Необходимость постоянной поддержки при изменениях сайта
- Зависимость от API сторонних сервисов для AI-обработки
- Ограниченный размер рынка (только пользователи Ozon + Chrome)
