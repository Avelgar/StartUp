# Шаг 10. Приоритизация рисков

## ТОП-3 риска

### Риск 1: Недостаточная точность

**Критичность:** высокая  
**Проверка:** benchmark vs Kinect  
**Метрика успеха:** ≥80–90% удовлетворённости  

---

### Риск 2: Realtime производительность

**Гипотеза:** latency приемлема для интерактива  

**Проверка:**

- измерение FPS  
- end-to-end latency  
- user testing  

**Критерий:**

- ≥30 FPS  
- latency <100–150 ms  

---

### Риск 3: Ценность SDK

**Гипотеза:** разработчики не хотят сами собирать MediaPipe.

**Проверка:**

- GitHub interest  
- количество установок  
- интервью  

**Критерий:**

- ≥30% говорят «сильно упрощает жизнь»
