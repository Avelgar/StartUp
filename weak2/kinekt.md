# Проект «Kinect» — Неделя 2

## Задачи недели

Анализ ситуации рынка. Формирование гипотезы бизнес-модели для проработки на протяжении программы.

---

## Шаг 1. Фиксация идеи и проблемного контекста

### Идея продукта

Локальная система компьютерного зрения для Windows, позволяющая отслеживать движения тела человека в реальном времени через обычную веб-камеру и использовать их для управления приложениями, играми и интерактивными системами.

---

### Проблема

После ухода Kinect с массового рынка разработчики и энтузиасты лишились доступного и удобного инструмента для body tracking.

Существующие решения:

- требуют специализированного оборудования  
- сложны в интеграции  
- избыточны по цене  
- не ориентированы на desktop-разработчиков  

При этом спрос на gesture-интерфейсы, интерактив и CV-решения продолжает расти.

---

### Сценарий

**Пользователь/роль:** разработчик CV / гейм-разработчик / технический энтузиаст

**Ситуация:** пользователь хочет добавить body tracking или gesture control в своё приложение или прототип.

**Проблема:**

- Kinect снят с производства  
- hardware-решения дорогие  
- OpenPose тяжёлый и сложный  
- MediaPipe — это только low-level библиотека  
- нет готового Windows-продукта «из коробки»

**Последствия:**

- **Время:** недели на интеграцию CV-стека  
- **Деньги:** $200–$1000 на оборудование  
- **Качество:** нестабильные самописные решения  
- **Риски:** высокий порог входа для экспериментов

### Сегмент 1: Разработчики CV и интерактивных систем (приоритет)

**Кто принимает решение:** сам разработчик  
**Кто пользователь:** CV-разработчики, HCI-исследователи, технические энтузиасты  

**Проявление проблемы:**

**Ограничения:**

## Шаг 7. Формирование ценностного предложения

### Формулировка

## Шаг 10. Приоритизация рисков

### ТОП-3 риска

**Риск 1:** Недостаточная точность

**Критичность:** высокая  
**Проверка:** benchmark vs Kinect  
**Метрика успеха:** ≥80–90% удовлетворённости  

---

**Риск 2:** Realtime производительность

**Гипотеза:** latency приемлема для интерактива  

**Проверка:**

- измерение FPS  
- end-to-end latency  
- user testing  

**Критерий:**

- ≥30 FPS  
- latency <100–150 ms  

---

**Риск 3:** Ценность SDK

**Гипотеза:** разработчики не хотят сами собирать MediaPipe.

**Проверка:**

- GitHub interest  
- количество установок  
- интервью  

**Критерий:**

- ≥30% говорят «сильно упрощает жизнь»
